# КРИТИЧЕСКОЕ ПРАВИЛО

В presentation-слое всех `*-impl` модулей использовать **ТОЛЬКО MVVM**.
**TEA/MVI ЗАПРЕЩЕН**, даже если он встречается в существующем коде (`phone_validation` — легаси, не образец).
Эталон в репо — `RegistrationViewModel` + `RegistrationComponentImpl`.
Подробности — `references/presentation-architecture.md`.

# Работа со skill

Всегда СТРОГО следуй принятой архитектуре в проекта и не отступай от нее.
Перед написанием кода читай соответствующий файл из `references/`:
- `presentation-architecture.md` — MVVM, ViewModel, Component, Screen
- `clean-architecture.md` — слои, направление зависимостей
- `module-structure.md` — раскладка `*-api` / `*-impl`
- `di.md` — Metro DependencyGraph

При объяснении кода всегда включай:

1. **Начни с аналогии**: Сравни код с чем-то из повседневной жизни
2. **Нарисуй диаграмму**: Используй ASCII-арт, чтобы показать поток, структуру или взаимосвязи
3. **Пройдись по коду**: Объясни шаг за шагом, что происходит
4. **Подсвети подвох**: Какая распространённая ошибка или заблуждение?

Излагай объяснения в разговорной форме. Для сложных концепций используй несколько аналогий.
