# Документирование кода

Этот skill описывает правила документирования кода в проекте с использованием KDoc.

## Основные принципы

1. **Все публичные элементы должны быть задокументированы** - классы, интерфейсы, функции, свойства
2. **Каждый модуль должен иметь README.md** - с описанием назначения и использования
3. **Используй KDoc** - стандартный формат документации для Kotlin
4. **Документация на русском языке** - для единообразия с проектом

## Дополнительные ресурсы

- Для понимания как писать KDoc комментарии, смотри [kdoc-guidelines.md](references/kdoc-guidelines.md)
- Для понимания как создавать README для модулей, смотри [module-readme.md](references/module-readme.md)
- Для примеров документации в проекте, смотри [examples.md](references/examples.md)

## Чеклист документирования

### При создании нового модуля
- [ ] Создать README.md в корне модуля
- [ ] Описать назначение модуля
- [ ] Добавить примеры использования
- [ ] Указать зависимости модуля

### При добавлении публичного API
- [ ] Добавить KDoc к классу/интерфейсу
- [ ] Документировать все публичные методы
- [ ] Документировать публичные свойства
- [ ] Добавить `@param`, `@return`, `@throws` где необходимо
- [ ] Добавить примеры кода через `@sample` или code blocks

### При ревью документации
- [ ] Проверить актуальность описаний
- [ ] Убедиться в наличии примеров использования
- [ ] Проверить корректность `@param` и `@return`
- [ ] Обновить README если изменился API

## Быстрый старт

### Документирование класса
```kotlin
/**
 * Компонент экрана авторизации по номеру телефона.
 *
 * Отвечает за валидацию номера телефона и отправку SMS-кода.
 * Реализует [UiComponent] для интеграции с Compose UI.
 *
 * @see PhoneValidationComponent.Factory для создания экземпляра
 */
interface PhoneValidationComponent : UiComponent {
    // ...
}
```

### Документирование функции
```kotlin
/**
 * Валидирует номер телефона по маске.
 *
 * @param phone номер телефона в формате +7XXXXXXXXXX
 * @return `true` если номер валиден, `false` в противном случае
 * @throws IllegalArgumentException если [phone] пустой
 */
fun validatePhone(phone: String): Boolean
```
